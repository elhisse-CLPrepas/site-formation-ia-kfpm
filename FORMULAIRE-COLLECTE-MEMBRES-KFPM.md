# Formulaire de collecte des membres - Formation IA KFPM

## Objectif

Ce formulaire sert a identifier les membres de l'equipe qui participeront a la formation IA KFPM et a preparer leur acces a la page GitHub Pages :

```text
https://elhisse-clprepas.github.io/site-formation-ia-kfpm/
```

Les informations collectees permettent de :

- confirmer l'identite du membre ;
- preciser son appartenance a KPM, FPM ou KFPM ;
- definir son role dans l'atelier formation IA ;
- enregistrer son email d'acces ;
- organiser les groupes de formation ;
- suivre les besoins et le niveau initial.

## Regle importante

Ce formulaire rempli contient des informations personnelles. Il ne doit pas etre publie dans le depot public ni sur GitHub Pages.

Utilisation recommandee :

1. remplir le formulaire ;
2. transmettre la version completee au responsable formation ;
3. inviter le compte GitHub dans le depot prive et partager Drive avec l'email valide ;
4. conserver les formulaires remplis dans un espace prive.

## Procedure de connexion a la page GitHub Pages

### Etape 1 - Recevoir le lien

Le responsable communique le lien :

```text
https://elhisse-clprepas.github.io/site-formation-ia-kfpm/
```

### Etape 2 - Utiliser l'email autorise

Le membre saisit l'email qui a ete declare dans le formulaire.

Exemples autorises selon la configuration actuelle :

- un email precise dans `assets/js/config.js` ;
- un email appartenant a un domaine autorise ;
- un email ajoute par le responsable formation.

### Etape 3 - Acceder au contenu

Si l'email est reconnu, les sections de formation deviennent visibles :

- modules essentiels ;
- methode LN-IA ;
- ressources ;
- supports et documents.

### Etape 4 - Demander une autorisation

Si l'email n'est pas reconnu, le membre doit contacter le responsable formation avec :

- nom et prenom ;
- appartenance ;
- role ;
- email a autoriser ;
- besoin de formation.

## Procedure responsable formation

### 1. Verifier le formulaire

Verifier :

- identite ;
- appartenance ;
- role ;
- email ;
- autorisation responsable ;
- besoin de formation.

### 2. Ouvrir les acces prives

Demander le nom d'utilisateur GitHub, inviter ce compte dans le depot prive avec le role minimal, puis partager les supports Drive directement avec l'email valide. Ne jamais publier la liste des emails dans GitHub Pages.

### 3. Mettre a jour le site

Apres modification :

```bash
git add .
git commit -m "Update authorized training members"
git push
```

### 4. Informer le membre

Envoyer le lien GitHub Pages et rappeler :

- utiliser l'email autorise ;
- ne pas partager les supports non valides ;
- demander de l'aide si l'acces ne fonctionne pas.
